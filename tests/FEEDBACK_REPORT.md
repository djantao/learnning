# MindForge E2E 工作流测试反馈报告

> 测试日期: 2026-05-10 | 测试文件: `tests/e2e-workflows.mjs`
> 结果: 43 项检查 | ✅ 38 通过 | ❌ 5 失败 (88.4% 通过率)

---

## 一、各工作流测试结论

### 工作流 1: 笔记 → 闪卡 → 复习闭环
**结论: ⚠️ 核心链路断裂**

| 步骤 | 结果 | 说明 |
|------|------|------|
| 创建笔记本/章节/笔记 | ✅ | 链路正常 |
| 笔记标签关联 | ✅ | 标签正确创建并关联 |
| Wiki 链接解析 | ⚠️ | API 返回 `linksFrom`/`linksTo`，非直观的 `outLinks`/`inLinks` |
| AI 生成闪卡 | ✅ | AI 调用正常 |
| **闪卡进入复习队列** | **❌** | **关键BUG：创建闪卡时 `sm2NextReview` 未设默认值，卡片永远不进入队列** |
| 复习会话 UI | ✅ | 页面渲染正常 |

**根本原因:** `POST /api/flashcards` 创建时不设 `sm2NextReview`，数据库为 NULL。`GET /api/review/due` 过滤条件 `sm2NextReview: { lte: new Date() }` 不匹配 NULL 值。

**修复方案:** 在 `src/app/api/flashcards/route.ts` POST handler 的 `create` data 中加入 `sm2NextReview: new Date()`。

目标树创建 → 子目标 → 材料关联 → 进度计算 → 标记完成 → 父目标级联，全部通过。

## 二、发现的缺陷

### 🔴 P0 — 阻断性缺陷

| # | 缺陷 | 影响 | 修复位置 |
|---|------|------|----------|
| 1 | **闪卡创建后 `sm2NextReview` 为空，复习队列永远为空** | 核心功能「间隔复习」完全不可用 | `src/app/api/flashcards/route.ts` POST handler，create data 中加入 `sm2NextReview: new Date()` |

### 🟡 P2 — 功能缺陷

| # | 缺陷 | 影响 | 说明 |
|---|------|------|------|
| 2 | 学习档案 GET 返回 `knowledgeLevel: undefined` | 学习档案配置无法被 AI 上下文正确使用 | 需排查 POST 存储和 GET 读取的数据一致性 |

### 🔵 P3 — 设计/体验改进

| # | 建议 | 说明 |
|---|------|------|
| 3 | 笔记详情 API 字段命名 | `linksFrom`/`linksTo` 不够直观，建议映射为 `outLinks`/`inLinks` |
| 4 | AI 生成闪卡失败时应有降级提示 | 当 AI 不可用时，前端无明确提示 |
