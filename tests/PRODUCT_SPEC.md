# MindForge 产品功能说明书

> 版本: 原型 v0.1 | 更新: 2026-05-10
> MindForge 是一个 AI 驱动的知识管理 + 间隔重复学习平台。

---

## 一、功能全景

| 编号 | 功能模块 | 核心价值 |
|------|----------|----------|
| F1 | 笔记系统 | Markdown 笔记 + Wiki 链接 + 标签组织 |
| F2 | 笔记本/章节 | 多层目录结构组织笔记 |
| F3 | AI 对话 | 带持久记忆的 AI 聊天（档案/锚点/盲区） |
| F4 | 课程系统 | 结构化课程 → 模块 → 知识点 + AI 教练 |
| F5 | 间隔复习 | SM-2 算法驱动的闪卡复习 |
| F6 | 学习目标 | 可嵌套目标树 + 自动进度计算 |
| F7 | 知识图谱 | 笔记间 Wiki 链接的力导向可视化 |
| F8 | 全文搜索 | 笔记内容全文搜索 + 标签过滤 |
| F9 | 学习档案 | 用户知识水平/偏好/指令锚点配置 |
| F10 | 活动追踪 | 每日学习统计数据 + 连续天数 |

---

## 二、核心用户流程

### 流程 A：笔记记录 → 闪卡生成 → 复习闭环

```
创建笔记本 → 创建章节 → 写笔记(Markdown+标签+Wiki链接)
    → AI 生成闪卡 → AI 生成摘要
    → 笔记出现在知识图谱中
    → 闪卡进入复习队列 → SM-2 间隔复习 → 掌握
```

**涉及功能点：** F1, F2, F5
**关键 API：** `POST /api/notes` → `POST /api/ai/generate-flashcards` → `POST /api/review/grade`
**关键数据流：** 笔记内容 → AI 生成闪卡 → SM-2 状态机 → 活动追踪

### 流程 B：课程学习 + AI 教练辅导

```
创建课程 → 添加模块(3层) → 添加知识点
    → 进入学习视图(分屏：内容+AI聊天)
    → 自我评估掌握度(1-5星)
    → AI 教练对话(含课程上下文)
    → AI 生成复习内容(闪卡/费曼问题)
    → AI 评估回答 → 掌握度自动更新 → 模块进度级联
```

**涉及功能点：** F4, F3
**关键 API：** `POST /api/courses` → `POST /api/ai/chat`(with courseId/kpId) → `POST /api/ai/review/evaluate`
**关键数据流：** 课程结构 → AI 上下文注入 → 掌握度变更 → 树形进度传播

### 流程 C：目标设定 → 材料关联 → 自动进度

```
创建目标树(可嵌套) → 关联笔记/闪卡为学习材料
    → 系统自动计算进度(基于闪卡掌握率)
    → 复习闪卡时进度实时更新
    → 标记材料完成
```

**涉及功能点：** F6, F5
**关键 API：** `POST /api/goals` → `POST /api/goals/[id]/materials` → `GET /api/goals/[id]`(实时进度)
**关键数据流：** 闪卡评分 → recomputeGoalsForPage → 目标树进度级联

### 流程 D：配置学习档案 → AI 个性化

```
设置学习档案(知识水平/目标/风格/语言偏好)
    → 配置指令锚点(硬规则，如"用中文解释")
    → 每次 AI 对话自动注入档案+锚点
    → AI 检测知识盲区 → 更新盲区列表 → 下次对话提醒
```

**涉及功能点：** F9, F3
**关键 API：** `POST /api/profile` → `POST /api/anchors` → `POST /api/ai/detect-blind-spots`
**关键数据流：** 档案/锚点 → context-builder → 系统提示词 → AI 响应个性化

### 流程 E：搜索 → 标签 → 图谱 知识发现

```
全文搜索笔记 → 按标签过滤浏览 → 在知识图谱中可视化笔记关联
```

**涉及功能点：** F7, F8, F1
**关键 API：** `GET /api/notes?search=` → `GET /api/tags` → `GET /api/graph`

## 四、模块间数据依赖

```
笔记 ←→ 闪卡 (pageId 关联)
笔记 ←→ Wiki链接 (NoteLink 双向)
笔记 ←→ 标签 (PageTag 多对多)
笔记 ←→ 笔记本/章节
闪卡 → 复习日志 → SM-2状态
闪卡 → 目标材料 → 目标进度
课程 → 模块 → 知识点 → 掌握度
知识点掌握度 → 模块进度 → 课程进度
复习评分 → 活动追踪
AI对话 → 上下文(档案+锚点+盲区+课程)
```

## 五、关键业务规则

1. **SM-2 算法：** 评分 >= 3 算正确，间隔增长；评分 < 3 重置为 1 天；EF 下限 1.3
2. **掌握度映射：** 知识点 0-5 星 → 0-100% 进度
3. **模块进度：** 叶子模块 = 知识点掌握度平均；父模块 = 子模块平均
4. **目标进度：** 基于关联闪卡的 SM-2 间隔（>=21天=100%, 7-21天=60%, 已复习=30%, 未复习=0%）
5. **自动状态切换：** 0%→not_started, 1-99%→in_progress, 100%→completed
6. **AI 上下文截断：** 笔记内容 2000 字符、知识点内容 3000 字符
7. **认证：** 原型阶段无密码，输入邮箱即登录/注册
